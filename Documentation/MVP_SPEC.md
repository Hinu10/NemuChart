# NemuChart MVP仕様

## 製品境界

NemuChartは、本人が入力した睡眠記録を振り返り、生活習慣づくりを支援するiOSアプリである。医療機器、診断支援、睡眠段階の測定器ではない。スコアは生活目標との比較、傾向は入力済みデータ内の関連候補として扱う。

## 12のMVP体験

1. 初回起動時に非医療の説明を読む。
2. 希望睡眠時間、標準起床時刻、週開始曜日を設定する。
3. 時間帯に応じたホーム画面を見る。
4. 朝に前夜のベッド時刻、入眠、起床、スッキリ度を入力する。
5. 任意で中途覚醒、スヌーズ、二度寝、昼寝、生活要因を入力する。
6. 内容を確認し、同じ睡眠日の重複なしに端末へ保存する。
7. 日次睡眠スコアと内訳を確認する。
8. 羊の穏やかなフィードバックを受け取る。
9. 今夜の目標を設定する。
10. 7日間の記録、週間スコア、分析信頼度を確認する。
11. 羊の元気度・成長、週間目標の進捗を見る。
12. 設定の変更、過去記録の編集・削除、全データ削除を行う。

## 画面フロー

`初回説明 → 初回設定 → ホーム → 朝の記録 → 確認 → 日次結果 → 今夜の目標 → ホーム`

ホームから、過去記録、週間ダッシュボード、週間目標、設定へ移動できる。夜間は記録開始を主操作にせず、端末を置くための穏やかな案内を表示する。

## 用語

- **睡眠日:** 起床した場所のローカル暦日。`yyyy-MM-dd`を一意キーとし、記録時のIANAタイムゾーンIDも保持する。
- **ベッド時刻:** 眠るためにベッドへ入った実時刻。
- **入眠時刻:** ユーザーが推定して入力した、眠り始めた実時刻。測定値ではない。
- **睡眠時間:** 入眠時刻から起床時刻までの実経過時間。
- **スッキリ度:** 起床時の主観的な感覚を1〜5で記録した値。
- **日次睡眠スコア:** 入力と個人目標を所定ルールで0〜100に換算した参考値。
- **目標達成度:** ユーザーが選んだ目標への進捗。睡眠スコアとは別の値。
- **分析信頼度:** 記録数や欠測量に基づく、傾向表示のためのデータ充足度。
- **元気度:** 直近状態に応じて変わる羊の一時的な状態。
- **成長:** 累積ポイントによる羊の段階。低スコアで失われない。

## 入力項目

必須項目は、睡眠日、ベッド時刻、入眠時刻または入眠潜時、起床時刻、スッキリ度である。

任意項目は、中途覚醒回数、スヌーズ回数、二度寝時間、昼寝時間、飲酒、カフェイン、スマートフォン終了時刻、ストレス、快適さである。任意項目は未入力と「0回・なし・摂取していない」を区別する。

## 分析条件

- 欠測を0として補完しない。
- 日次結果は入力済み要素のみで算出し、内訳とルールバージョンを保持する。
- 週間集計は対象日数と記録数を表示する。
- 傾向は最低サンプル数を満たす場合のみ表示し、因果関係として表現しない。
- データ不足はエラーではなく、明示的な値として扱う。

## 日次睡眠スコア v1.0.0

日次睡眠スコアは、睡眠時間40点、通常の起床時刻との近さ25点、主観的なスッキリ度25点、中途覚醒による分断10点の合計100点とする。睡眠時間は希望時間との差が4時間で0点、起床時刻は通常時刻との差が3時間で0点となるよう線形に換算する。スッキリ度は5段階を0〜満点へ、中途覚醒は0回を満点、5回以上を0点へ換算し、各点は四捨五入する。

中途覚醒が未入力の場合、その10点は睡眠時間・起床時刻・スッキリ度へ44点・28点・28点となるよう再配分する。未入力を0回として扱わない。飲酒、カフェイン、ストレスなどの生活要因は機械的な減点に使わない。目標達成度は別指標とし、スコアは医療上の睡眠評価ではない。計算結果は内訳とルールバージョンを伴い、同じ入力と同じルールから同じ値を返す。

## 禁止表現

「診断」「治療」「完治」「測定したレム睡眠／ノンレム睡眠」「必ず改善する」「最適である」「病気である」など、医学的判断、測定精度、効果、因果を断定する表現を使用しない。「入力からの推定」「傾向の可能性」「参考値」「相談を検討してください」を用いる。

## MVP対象外

独自アラーム、HealthKit、Apple Watch、睡眠段階推定、CloudKit同期、CSV/JSON書き出し、月間・曜日別分析、課金、アカウント、独自バックエンド、医療機関検索、医療レポートを含めない。
